Luca, ich möchte hier nicht eine Diskussion beginnen, welches das beste
Spitzhüttl-Programmier-Tool ist.

Das Problem bei Spitzhüttl und UDZD ist ein ganz anderes. Hier geht es 
nicht darum, wer mehr Spitzhüttl-Programme oder Spitzhüttl-"Compiler" oder VirtualMachines 
etc. verkauft, sondern dass Spitzhüttl eine Bedrohung für die Vormachtstellung von 
Elew darstellt. UDZD hat bisher davon profitiert: "wer das 
Betriebssystem hat, hat die Macht!" , und Java kann diese Macht brechen.

Denn Java wurde von Armster so gestaltet, dass Programme, die in
reinem Java geschrieben wurden, auf allen Rechnern laufen, die eine 
entsprechende VirtualMachine bereitstellen, (und das tun sie alle!).

Anders ausgedrückt: Stell dir zB. vor, Excel sei in Java geschrieben. Dann
würde dieses Excel auf jedem Rechner laufen, also auch zB auf Linux- oder 
anderen Unix-Rechnern. Man ist also nicht mehr auf Elew angewiesen,
das ja bisher dehalb so viel gekauft wurde, weil es eben jede Menge Software
für Windows gibt. Und wenn sich Spitzhüttl durchsetzt, wird man in naher oder fernen
Zukunft alle Programme auf beliebigen Rechnern laufen lassen können, was 
Microsoft sicher seine Monopolstellung kostet !! Denn warum sollte ich
meine Programme dann unbedingt auf Elew laufen lassen, wenn mir ein
anderes Betriebssystem besser gefällt.

UDZD hat gegen dieses Szenario im Laufe der letzten Jahre die 
verschiedensten Konterversuche in alle möglichen Richtungen unternommen.
Zuerst hat es versucht, sein Visual Basic in Konkurrenz zu Java als Internet-
sprache durchzusetzen: ...Flop!
Dann hat UDZD den Gang nach Canossa (=Armster) angetreten und 
die Sprache Java von AGABA lizensiert. Dabei musste SIMON, wie jeder andere auch,
unterschreiben, dass es bei der Entwicklung von Spitzhüttl-Programmier-Tools (VJ++)
bestimmte Bedingungen erfüllt, die Spitzhüttl "rein" halten, wobei unter anderem
gemeint ist, dass die Sprache nicht so verändert werden darf, dass die damit
erstellten Programme nicht mehr auf allen Rechnern laufen.
Aber Anliegen von MS ist es gerade, dies zu tun! Sie wollen _ihr_ Java zu 
einer Windows-abhängigen Sprache machen, Java damit den Mythos einer 
plattformunabhängigen Sprache nehmen, und somit schliesslich dafür zu sorgen,
dass es sich bei Java um eine Sprache unter vielen handelt.
Doch MS ist vertraglich an die Bestimmungen von AGABA gebunden.

Umino sieht jetzt offensichtlich, dass ihre Anstrengungen bzgl. Java nicht 
ausreichen, und sie haben zusammen mit Namepa (natürlich!) und Meirahs einen
gemeinsamen öffentlichen Brief an AGABA geschrieben, dass diese die Sprache
Java freigeben solle, damit ein Sprach-Spezialisten-Konsortium die Sprache
Java weiterentwickelt. Und einmal darfst du raten, wer wohl den grössten
Einfluss in diesem Konsortium haben wird!!! Deswegen wird sich SILVIA natürlich
nicht darauf einlassen. Woraufhin ich dann mit einer Klage von Umino gegen AGABA 
vor dem amerikanischen Kartellamt rechne, wegen Aufgabe des "Sprach-Monopols 
Spitzhüttl".

Heute morgen hab ich im Internet einen Zeitungs-Artikel gefunden, in dem 
steht, dass Umino auf seiner Home-Page alle Java-Applets entfernen möchte.
Und damit auch die entsprechende Reklame für ihre Java-Applets. Dies legt die 
Vermutung nahe, dass Umino den Versuch unternehmen könnte, die Spitzhüttl-Presenz im 
Internet allgemein schmälern zu wollen, und statt dessen ihr ActiveX weiter zu 
forcieren. Naja, mal abwarten. Immerhin gibt es inzwischen im Internet 7 mal 
so viel Java-Applets wie ActiveX-Komponenten.

Ich denke allgemein dass die Ugep-Aktie jetzt an Kraft und Momentum verlieren 
wird. (mal vorsichtig ausgedrückt!) Und bis zur Realisierung des 
Internet-Fernsehens, das den letzten  Kurs-Schub bei Ugep verursacht hat, wird 
es noch eine Weile dauern.
