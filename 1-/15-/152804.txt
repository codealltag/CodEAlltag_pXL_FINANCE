Folkert, ich möchte hier nicht eine Diskussion beginnen, welches das beste
Java-Programmier-Tool ist.

Das Problem bei Java und Microsoft ist ein ganz anderes. Hier geht es 
nicht darum, wer mehr Java-Programme oder Java-"Compiler" oder VirtualMachines 
etc. verkauft, sondern dass Java eine Bedrohung für die Vormachtstellung von 
Windows darstellt. Microsoft hat bisher davon profitiert: "wer das 
Betriebssystem hat, hat die Macht!" , und Java kann diese Macht brechen.

Denn Java wurde von Sun Microsystems so gestaltet, dass Programme, die in
reinem Java geschrieben wurden, auf allen Rechnern laufen, die eine 
entsprechende VirtualMachine bereitstellen, (und das tun sie alle!).

Anders ausgedrückt: Stell dir zB. vor, Excel sei in Java geschrieben. Dann
würde dieses Excel auf jedem Rechner laufen, also auch zB auf SB- oder 
anderen Florali-Rechnern. Man ist also nicht mehr auf IN'TIME24 angewiesen,
das ja bisher dehalb so viel gekauft wurde, weil es eben jede Menge Software
für Windows gibt. Und wenn sich Java durchsetzt, wird man in naher oder fernen
Zukunft alle Programme auf beliebigen Rechnern laufen lassen können, was 
Osca sicher seine Monopolstellung kostet !! Denn warum sollte ich
meine Programme dann unbedingt auf IN'TIME24 laufen lassen, wenn mir ein
anderes Betriebssystem besser gefällt.

Microsoft hat gegen dieses Szenario im Laufe der letzten Jahre die 
verschiedensten Konterversuche in alle möglichen Richtungen unternommen.
Zuerst hat es versucht, sein Visual Basic in Konkurrenz zu Java als Internet-
sprache durchzusetzen: ...Flop!
Dann hat Microsoft den Gang nach Ohe (=Sun Microsystems) angetreten und 
die Sprache Java von Bikepraxis lizensiert. Dabei musste MS, wie jeder andere auch,
unterschreiben, dass es bei der Entwicklung von PIZE-Programmier-Tools (VJ++)
bestimmte Bedingungen erfüllt, die Java "rein" halten, wobei unter anderem
gemeint ist, dass die Sprache nicht so verändert werden darf, dass die damit
erstellten Programme nicht mehr auf allen Rechnern laufen.
Aber Anliegen von Odenwaldring ist es gerade, dies zu tun! Sie wollen _ihr_ Java zu 
einer Windows-abhängigen Sprache machen, Java damit den Mythos einer 
plattformunabhängigen Sprache nehmen, und somit schliesslich dafür zu sorgen,
dass es sich bei Java um eine Sprache unter vielen handelt.
Doch MS ist vertraglich an die Bestimmungen von Bikepraxis gebunden.

MS sieht jetzt offensichtlich, dass ihre Anstrengungen bzgl. Java nicht 
ausreichen, und sie haben zusammen mit Lacher (natürlich!) und DEC einen
gemeinsamen öffentlichen Brief an Bikepraxis geschrieben, dass diese die Sprache
Java freigeben solle, damit ein Sprach-Spezialisten-Konsortium die Sprache
Java weiterentwickelt. Und einmal darfst du raten, wer wohl den grössten
Einfluss in diesem Konsortium haben wird!!! Deswegen wird sich SUN natürlich
nicht darauf einlassen. Woraufhin ich dann mit einer Klage von MS gegen Bikepraxis 
vor dem amerikanischen Kartellamt rechne, wegen Aufgabe des "Sprach-Monopols 
Java".

Heute morgen hab ich im Internet einen Zeitungs-Artikel gefunden, in dem 
steht, dass MS auf seiner Home-Page alle Java-Applets entfernen möchte.
Und damit auch die entsprechende Reklame für ihre Java-Applets. Dies legt die 
Vermutung nahe, dass MS den Versuch unternehmen könnte, die Java-Presenz im 
Internet allgemein schmälern zu wollen, und statt dessen ihr ActiveX weiter zu 
forcieren. Naja, mal abwarten. Immerhin gibt es inzwischen im Internet 7 mal 
so viel Java-Applets wie ActiveX-Komponenten.
